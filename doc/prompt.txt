3. Core Architecture: The "Autonomous Loop"
Phase 0: The Data Lake (Initialization)
Action: On startup, check for sp500_history.duckdb.

If Missing: Trigger a Bulk Download of the top 50 S&P 500 stocks for the last 10 years.

Storage: Store as a localized Parquet/DuckDB file for instant querying (0ms latency).

Phase 1: The "Scout Agent" (Query Generator)
UI: User types a natural language goal: "Find me high growth tech stocks that are currently not overbought."

Logic: LLM translates this to a Pandas query against the local DB:

Sector == 'Technology'

Price < 50_Day_MA (Beaten down)

Revenue_Growth > 15% (Aggressive Growth)

Output: Returns Top 30 candidate tickers.

Phase 2: The "Evaluation Agent" (Risk & Sentiment)
Input: The 30 candidates.

Logic:

Technical Check: Filter by RSI < 70 (not overbought).

Risk Check (Proxy): Fetch ticker.news. If headlines contain ["lawsuit", "investigation", "misses estimates"], apply a "Risk Penalty".

Decision: Select the Top 10 stocks.

Phase 3: The "Portfolio Manager" (Execution)
Logic: Construct a portfolio.

Weighting: Equal Weight (10% each).

Output: A JSON structure of the "Buy Orders".

Phase 4: The "Time Machine" Backtester (The Judge Pleaser)
Trigger: "Run Backtest" button.

Algorithm:

Set Simulated_Date to 3 years ago.

Loop (Step = 6 Months):

Mask data to hide the future.

Run Scout -> Eval -> Portfolio Agents.

"Buy" the selected stocks.

Hold for 6 months.

Sell and Re-run Agents.

Result: Generate a dataset matching the Portfolio Cumulative Return vs SPY (Benchmark).

Fake the News: yfinance news is sparse. If news is empty, assume "Neutral Sentiment" to prevent crashes.
